# APIモジュール — ローカルコンテキスト

## 責務
- HTTPルート定義
- リクエストのバリデーションとパース
- レスポンスのフォーマット
- ミドルウェアチェーン（認証、ログ、エラーハンドリング）

## 主要ルール
1. すべてのエンドポイントでスキーマバリデーション（pydantic等）を使って入力を検証する
2. 統一されたエラーレスポンス形式を返す: `{"error": {"code": ..., "message": ...}}`
3. 適切なHTTPステータスコードを使用する
4. すべてのエンドポイントにdocstringでドキュメントを記述する
5. 明示的にパブリックでない限り、認証ミドルウェアでルートを保護する

## ルート命名規則
- 複数形の名詞を使用: `/users`, `/subscriptions`
- kebab-caseを使用: `/payment-methods`
- 関連リソースをネスト: `/users/:id/subscriptions`

## エラーコード
| ステータス | 用途 |
|-----------|------|
| 400 | 不正な入力 |
| 401 | 未認証 |
| 403 | 権限なし |
| 404 | リソースが見つからない |
| 409 | 競合（重複等） |
| 500 | 予期しないサーバーエラー |
