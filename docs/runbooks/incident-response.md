# インシデント対応手順書

## 深刻度レベル

| レベル | 説明 | 対応時間 |
|--------|------|---------|
| Critical | サービスダウン、データ損失リスク | 即時 |
| High | 主要機能の障害 | 1時間以内 |
| Medium | 軽微な機能の劣化 | 4時間以内 |
| Low | 見た目・緊急でない問題 | 次のスプリント |

## 対応手順

1. **評価** — 深刻度と影響範囲を判定する
2. **連絡** — 関係者に通知する
3. **緩和** — 可能であれば一時的な対処を適用する
4. **調査** — 根本原因を特定する
5. **修正** — 恒久的な修正を実装・デプロイする
6. **振り返り** — ポストインシデントレビューを行い、ドキュメントを更新する
