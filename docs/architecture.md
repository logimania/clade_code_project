# システムアーキテクチャ

## 概要

本プロジェクトは関心の分離を明確にしたモジュラーアーキテクチャを採用しています。

```
┌─────────────────────────────────────────┐
│          APIレイヤー (src/api/)          │
│      ルーティング、ミドルウェア、バリデーション │
├─────────────────────────────────────────┤
│          サービスレイヤー                 │
│      ビジネスロジック＆オーケストレーション  │
├──────────────┬──────────────────────────┤
│  認証モジュール │    課金モジュール        │
│  (src/auth/) │    (src/billing/)        │
├──────────────┴──────────────────────────┤
│        データアクセスレイヤー             │
│       リポジトリ＆モデル                  │
├─────────────────────────────────────────┤
│           データベース                   │
└─────────────────────────────────────────┘
```

## モジュールの責務

### APIレイヤー (`src/api/`)
- HTTPリクエストの処理とルーティング
- 入力バリデーションとシリアライゼーション
- 認証ミドルウェア
- エラーレスポンスのフォーマット

### 認証モジュール (`src/auth/`)
- ユーザー認証（ログイン、サインアップ、ログアウト）
- セッション管理
- トークン生成と検証
- パスワードのハッシュ化と検証

### 課金モジュール (`src/billing/`)
- サブスクリプション管理
- 決済処理
- 請求書生成
- 使用量トラッキング

## データフロー

1. リクエストがAPIレイヤーに到達
2. ミドルウェアが認証とバリデーションを処理
3. サービスレイヤーがビジネスロジックをオーケストレーション
4. データレイヤーがデータの永続化/取得を実行
5. レスポンスがAPIレイヤーを通じて返却
