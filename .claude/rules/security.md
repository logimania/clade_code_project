# セキュリティルール

## シークレット管理
- APIキー、パスワード、トークンをソースコードにハードコーディングしない
- .env ファイルをコミットしない
- すべてのシークレットは環境変数で管理する
- 機密データ（トークン、パスワード、個人情報）をログに出力しない

## 保護ディレクトリ
以下のディレクトリの変更にはユーザーの明示的な承認が必要:
- `src/auth/` — 認証・認可ロジック
- `src/billing/` — 決済処理・財務データ
- `migrations/` — データベーススキーマ変更

## 入力バリデーション
- システム境界でユーザー入力を必ず検証・サニタイズする
- APIリクエストボディにはスキーマバリデーション（zod, joi 等）を使用
- パラメータ化クエリを使用する — SQL文字列の結合は禁止
- XSS防止のためHTML出力をエスケープする
