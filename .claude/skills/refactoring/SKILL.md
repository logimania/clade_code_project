$ARGUMENTS を以下のプレイブックに従ってリファクタリングしてください。

## 開始前
1. 対象ファイルを読み、現在の動作を理解する
2. `npm run test` を実行し、変更前にすべてのテストが通ることを確認
3. リファクタリングの範囲を特定する

## リファクタリングパターン（必要に応じて適用）

### 関数の抽出
- いつ: 大きな関数内で1つの明確な処理を行うブロックがあるとき
- 方法: そのブロックを明確なパラメータと戻り値の型を持つ名前付き関数に移動

### 条件分岐の簡素化
- いつ: ネストされた if/else や複雑なブーリアン式があるとき
- 方法: 早期リターン、ガード句、または適切な名前のブーリアン変数への抽出

### 重複の除去
- いつ: 同じロジックが3箇所以上に存在するとき
- 方法: 共有ユーティリティに抽出（ロジックが本当に同一であることを確認した上で）

### 明確な命名への変更
- いつ: 変数名・関数名が曖昧または誤解を招くとき
- 方法: *どう動くか*ではなく*何を表すか*を説明する名前を選ぶ

## リファクタリング後
1. `npm run test` を実行 — 何も壊れていないことを確認
2. `npm run format` を実行してフォーマットを適用
3. 何を変更し、なぜ変更したかを要約
