以下のバグを調査・修正してください: $ARGUMENTS

## ステップ1: 再現
- バグを再現する正確な手順を特定する
- 期待される動作と実際の動作を記録する
- バグが常に発生するか、断続的かを確認する

## ステップ2: 切り分け
- 原因となるモジュール/ファイルを絞り込む
- `git log --oneline -20` で最近の変更を確認し、原因の可能性を調べる
- 実行フローを追跡するためのログを追加する

## ステップ3: 診断
- エラーメッセージとスタックトレースを注意深く読む
- 入力データに想定外の値がないか確認する
- 外部依存（API、DB等）に関する前提を検証する
- 環境固有の問題（環境変数、設定、バージョン）を確認する

## ステップ4: 修正
- 根本原因を修正するための最小限の変更を行う
- 症状ではなく原因を修正する
- 同じバグが他の箇所にも存在しないか検討する

## ステップ5: 検証
- `npm run test` を実行してリグレッションを確認する
- このバグを検出できるテストを追加する
- 修正に関連するエッジケースをテストする

## ステップ6: 記録
- 何が問題で、なぜ起きて、どう修正したかを要約する
- **`CLAUDE_ISSUE.md` にイシューを追記する**（症状、原因、解決策、再発防止）
- `CLAUDE_PROGRESS.md` に行った変更を記録する
