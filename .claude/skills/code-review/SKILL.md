$ARGUMENTS を以下のチェックリストに従ってコードレビューしてください。

## 正確性
- [ ] ロジックが正しく、エッジケースを処理している
- [ ] オフバイワンエラーがない
- [ ] null/undefined が適切に処理されている
- [ ] エラーパスが適切にハンドリングされている

## セキュリティ
- [ ] シークレットや認証情報がハードコーディングされていない
- [ ] 入力が検証・サニタイズされている
- [ ] SQLインジェクション、XSS、コマンドインジェクションのリスクがない
- [ ] 認証・認可チェックが適切に実装されている

## パフォーマンス
- [ ] 不要なDBクエリやAPI呼び出しがない
- [ ] N+1クエリパターンがない
- [ ] 大量データがページネーションされている
- [ ] 高コストな処理が適切にキャッシュされている

## 保守性
- [ ] 明確な命名でコードが自己文書化されている
- [ ] 関数が単一責務になっている
- [ ] 抽象化すべき重複ロジックがない
- [ ] 複雑なロジックに説明コメントがある

## テスト
- [ ] 新機能に対応するテストがある
- [ ] エッジケースがカバーされている
- [ ] テストが決定的（フレーキーテストなし）
- [ ] モック/スタブが適切に使用されている

指摘事項を重大度別にグループ化して報告: Critical > High > Medium > Low
